船舶一维设计：从阻力到油耗

先别急着记公式。这个项目要做的，是把一艘大船“以目标速度持续航行”这件事，翻译成可以估算的推力、功率与燃油消耗。

核心问题：如果一艘 40 万吨级散货船以设定航速航行，主机需要输出多少功率？这会带来多少燃油消耗？

向下阅读：把复杂的船舶系统拆成一条连续的能量链。

01 / 项目真正要解决什么

用一个足够简洁的模型，建立“性能基线”

真实船舶的阻力、螺旋桨、主机和海况会彼此耦合。课程项目先采用一维近似，把最关键的因果关系串起来。这样，后续无论讨论节能、助推或推进方案，都有一个可以比较的基准。

1

船体在水中有多大？

尺寸和方形系数决定排水体积与湿表面积，也决定水对船体“黏住”的面积。

2

为了保持航速，要克服多大阻力？

先估摩擦阻力，再用系数补进船体形状的影响，得到一维模型中的总阻力。

3

阻力会变成多少燃油？

推力平衡给出轴功率，主机负载率决定单位发电量的油耗，最后得到基线燃油消耗。

02 / 一眼看懂逻辑主线

每个公式都在回答前一步留下的问题

不要把这看成五段独立计算。它其实是一条因果链：几何决定与水接触的尺度，阻力决定推力，推力决定功率，功率和负载率决定油耗。

  1. STEP 01

    船型几何

    L、B、T、CB 描述船有多大、浸在水里有多深。

  2. STEP 02

    湿表面积

    船壳与水接触的面积，是摩擦阻力的直接尺度。

  3. STEP 03

    船体阻力

    速度越高、接触面积越大，船需要克服的阻力越大。

  4. STEP 04

    推进功率

    稳态航行时，螺旋桨推力抵消阻力；力乘速度变成功率。

  5. STEP 05

    主机油耗

    主机负载率决定 SFOC，最终得到每小时的燃油消耗。

读图诀窍：任何一个参数变化，都沿箭头向后传播。航速变化尤其明显，因为它同时影响雷诺数、摩擦阻力和功率。

03 / 主要公式：知道“为什么算”，再记“怎么算”

五步推导，建立完整闭环

以下保留原文件中的一维计算骨架。符号不熟没关系，先抓住每一步的输入、输出和物理意义。

第一步
排水量与湿表面积
Awet = 1.7 LppT + ∇ / T∇ = LppB T CB

要回答：船体浸在水里的体积和与水接触的面积有多大？Awet 是后面摩擦阻力的“面积底盘”。

第二步
雷诺数与摩擦系数
CF = 0.075 / [log10(Re) − 2]²Re = UshipLWL / ν

要回答：水流绕过船体时，黏性效应有多强？Re 将“船有多长、跑多快、流体有多黏”合成一个无量纲数。

第三步
船体阻力
RH,y = k1RfRf = ½ρAwetU²CF

要回答：为了让船保持速度，水会“拉住”船多大的力？文件中将附体阻力和造波阻力简化为 0，再用 k1 修正摩擦阻力。

第四步
推进与主机功率
Pshaft = Fy,propUship / ηpropFy,prop = RH,y

要回答：螺旋桨必须给多大推力，轴上要提供多少功？稳态时推力与阻力平衡；效率小于 1，表示有一部分能量在推进过程中损失。

第五步
负载率与油耗
mfuel = SFOC × Penginexopt = Pengine / Pengine,design
SFOC = ax² + bx + c

要回答：主机在当前工况下每小时烧多少油？负载率 x 决定单位电量油耗，随后 mfuel = SFOC × Pengine。

一个关键提醒

功率不是“力”本身。抵抗相同的力，速度越高，每秒完成的做功越多，所以功率需求会快速上升。这正是船舶节能设计非常重视航速的原因。

05 / 读公式时先盯住这几类量

符号不是障碍，它们是不同层次的问题

符号代表什么在逻辑链中的位置
L, B, T, CB船长、船宽、吃水、方形系数输入：定义船型的几何尺度。
Awet船体湿表面积几何输出：决定水与船壳的接触面积。
Re, CF雷诺数、摩擦阻力系数流体性质：把尺度和速度转化为阻力计算中的系数。
RH,y船体阻力力的层次：外界要求推进器抵消的阻碍。
Pshaft, Pengine轴功率、主机功率能量层次：把“力”翻译成“每秒需要做多少功”。
x, SFOC, mfuel负载率、单位油耗、燃油消耗经济层次：把主机工况翻译为每小时的燃料需求。

06 / 带着这条主线继续做项目

后续方案，都是在这条链上施加影响

改船型或载况

先影响排水量和湿表面积，继而影响阻力。

改推进或增加助推

直接改变需要由主机提供的净推力与轴功率。

改运行策略

改变航速与主机负载率，最终反映为燃油消耗变化。

一分钟自测

如果航速提高，而船型和推进效率不变，最先直接变化的是哪一个量？它为什么会继续影响油耗？

最先直接变化的是雷诺数与摩擦阻力；阻力变大后，为保持稳态，螺旋桨需要更大推力。推力乘以更高的速度，又会使功率需求进一步增加，主机负载率和燃油消耗随之上升。

07 / 课程任务

用 Python 实现一个船舶一维性能估算器

请独立编写一个 Python 小软件。用户输入船体、运行与发动机参数后，程序应沿着本页的计算链完成估算，并清楚输出船体阻力、主机负载、功率与燃油消耗量。

A / 输入

读取参数

接收船型几何、航速、流体参数、推进效率和发动机额定功率等数据。

B / 计算

复现一维模型

依次计算湿表面积、雷诺数、摩擦阻力、总阻力、轴功率与负载率。

C / 输出

给出运行结果

以规范单位报告阻力、功率、负载率、SFOC 与每小时燃油消耗。

必须支持的输入

  • 船体参数：Lpp、LWL、B、T、CB
  • 运行参数：航速 Uship、海水密度 ρ、运动黏度 ν、船体修正系数 k1
  • 推进参数：推进效率 ηprop
  • 发动机参数：额定功率 Pengine,design，以及 SFOC 曲线系数 a、b、c

必须输出的结果

  • 船体阻力 RH,y：建议同时给出 N 与 MN
  • 轴功率 Pshaft：以 kW 或 MW 给出
  • 发动机负载率 x：以百分数给出，并提示是否超过 100%
  • 燃油消耗：输出 SFOC（g/kWh）及 mfuel（kg/h 或 t/h）

完成标准

  1. 保持计算链完整：不能跳过湿表面积、雷诺数、摩擦阻力或负载率等中间步骤。
  2. 把每一步封装成清晰的函数：例如 wet_area()、resistance()、shaft_power()、fuel_consumption()。
  3. 标明单位：输入、输出和终端显示都要写清单位，避免 N、kN、MW、kW 混用。
  4. 完成基准验证：用本页 Valemax 参数和 5.0 m/s 航速运行一次，并解释结果是否落在约 1.04 MN 阻力、6.90 MW 轴功率的数量级。

提交物：一个可直接运行的 .py 文件，以及简短说明文档（输入参数、计算方法、一次基准运行的结果截图或文本输出）。
